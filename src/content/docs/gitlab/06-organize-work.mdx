### Organize your work with issues and branches

We previously discussed how to work with [issues](/gitlab/plan-work) and [branches](/gitlab/parallel-topics).
Issues help you keep track of tasks, while branches allow you to work on different topics in parallel.
Combining these two features can significantly enhance the productivity of your team.

### Link branches to issues

In general, when you create a new branch for a specific task, it is a good practice to link it to the corresponding issue.
To link an issue to a branch, you can include the issue number in the branch name (e.g., `3-issue-name` for issue `#3`).
This way we clearly indicate which issue that branch is addressing.

There is also a more **automated way** to link branches to issues:

<Steps>
  1. Go to the page of the issue you want to work on.
  2. Look for an option to create a branch from that issue, often in the development-related section.
</Steps>

![Create branch from issue](src/assets/gitlab/create-issue-branch.png)

Clicking that option will open a dialog window where you can specify the name of the new branch.
By default, the branch name will often be created by using the issue number and title.
You can modify the branch name, if needed, but it is recommended to keep at least the issue number in the branch name for clarity.

<Steps>
  1. Leave the branch source set to `main`, unless you have a specific reason to start from a different branch.
  2. Confirm the creation of the branch.
  3. [Switch to the new branch](/gitlab/parallel-topics#switch-between-branched-out-topics) to start working on the issue.
</Steps>

![Dialog to create branch from issue](src/assets/gitlab/issue-branch-dialog.png)

### Closing an issue with a branch

Once you consider an issue to be completed and you are ready to [merge your branch](/gitlab/parallel-topics/#merge-branches), it is time to create a **merge request**.
It is a good practice to link the corresponding issue to the merge request, by adding it to the merge request's description.
To do that, you can simply write `#` followed by the issue number (e.g., `#3`) and the link will be automatically created.
If you also combine that with specific keywords then the issue can be **automatically closed** when the merge request is merged.
For example, you can write `Closes #3` or `Fixes #3` in the merge request description.
