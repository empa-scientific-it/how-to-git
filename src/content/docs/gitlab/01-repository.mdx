### What is a Git repository?

A Git repository is a folder that contains all the files and folders related to a project.
Beyond that, it also contains the history of all changes made to them.
Further in the tutorial, we will learn how to use Git to manage a repository: track changes, work on different topics in parallel, and collaborate with others, and more.

### Create a repository

To start the tutorial, we need to create a repository.
In GitLab, the repository usually lives inside a **project**.
That project will serve as a place to store our text file.

To create a repository, please go to your GitLab account and click on the button to create a new project.
The exact page layout may vary slightly, but the overall flow is the same.

In the newly opened page, please fill in the following fields:

<Steps>

1. Choose to create a **blank project**

2. Set the **project name** to `git-tutorial`

3. Set the **description** to `Git tutorial` (optional)

4. Make sure the **visibility** is set to `Public` if you want to follow the tutorial with the simplest sharing settings inside the GitLab instance

5. Leave the rest of the options set to default

6. Click on the button to `Create project`

</Steps>

Congratulations, you have created your first GitLab repository.
The project is empty, so we need to add a file to it.

### Public vs Private repositories

When creating a repository, you can choose whether it will be public or private.
This choice affects who can see the content of the repository and affects how you can collaborate with others.

- **Public project**: Anyone with access to the GitLab instance can see the content of the repository.
  In your case, this mainly means people in your organization who can sign in to that GitLab environment.
  This is useful when you want to share your work broadly inside the organization and allow others to report issues, suggest changes, and contribute.

- **Private project**: Only you and the people you invite can see the content of the repository.
  This is useful for personal projects or when you want to keep your work private.
  You can still invite people to collaborate on the repository, but they will need a GitLab account and access to the project.

Setting the visibility of the repository is the initial step in controlling access to content and collaboration features.
You can manage access to the repository in more detail later by inviting people with different roles.
Those details will be discussed in the [Manage access to the repository](/gitlab/collaborate/#manage-access-to-the-repository) section.

### Add a file to the repository

To add a file to the repository, use the option to create a new file or upload an existing one.
In the newly opened page, please either drag and drop a text file or create one directly in the browser.
You can create the file on your computer using any available text editor.
The file can contain any text you want, but usually programmers like to put "Hello, World!", so let's follow the tradition.
The filename does not matter.

Let's not modify any other fields and confirm the commit.
Congratulations, you have added a file to the repository.
